# my-blog

마크다운 파일을 읽어 정적 블로그 웹사이트로 변환하는 프로젝트입니다.

## 핵심 원칙

- **프레임워크 없음**: React, Vue, Next.js 등 사용하지 않음. 순수 HTML, CSS, JavaScript(Vanilla JS)로만 구현.
- 빌드 도구(Webpack, Vite 등)나 패키지 매니저 의존성은 꼭 필요한 경우가 아니면 추가하지 않음. 마크다운 파싱 라이브러리 정도만 최소한으로 허용.
- 정적 사이트: 서버 사이드 로직 없이 브라우저에서 동작하거나, 빌드 시점에 마크다운을 HTML로 변환하는 단순한 스크립트 형태.

## 디자인 요구사항

- 깔끔하고 가독성 좋은 타이포그래피 (본문 읽기에 최적화된 줄 간격, 폭 제한)
- 다크 모드 지원 (시스템 설정 감지 + 수동 토글, `prefers-color-scheme` 활용)
- 모바일 반응형 — 작은 화면에서도 레이아웃이 깨지지 않도록 미디어 쿼리로 대응
- 불필요한 장식 요소 배제, 콘텐츠(글) 중심의 미니멀한 UI

## 작업 시 주의사항

- 새 기능을 추가할 때 프레임워크나 빌드 체인을 끌어오려는 유혹을 피하고, 항상 "이걸 vanilla JS로 할 수 있는가"를 먼저 검토할 것.
- CSS는 다크 모드 변수(예: CSS custom properties)를 기준으로 색상을 관리하여 라이트/다크 전환이 쉽게 되도록 구성.
- 마크다운 파일 구조나 변환 스크립트의 위치가 정해지면 이 문서에 추가로 기록.
